M-Core RTOS

// A Real-Time Operating System built from scratch on ARM Cortex-M

A ground-up implementation of a preemptive real-time operating system targeting the STM32F411 Nucleo on ARM Cortex-M4. Implements a round-robin scheduler with SysTick-driven context switching, periodic callbacks and cooperative yielding layered on top of the same scheduler core, a thread control block subsystem, and binary semaphore synchronization, built directly on SysTick and NVIC hardware primitives.

System Architecture

The system is structured as four distinct layers. Each layer has a single responsibility and communicates only with the layer directly below it, a strict dependency hierarchy that mirrors production-grade embedded OS design.

layered architecture — hardware to application
L4 — Application Layer
User Tasks / Threads Application Logic Semaphore Consumers
L3 — RTOS Kernel
Round-Robin Scheduler Periodic Callback (layered) Cooperative Yield (layered) SysTick Context Switch TCB Manager Semaphore / IPC
L2 — Driver Layer
LED Driver (GPIO PA5) UART Driver (USART2 TX) Timebase (SysTick)

SysTick — scheduler + context switch TIM2 — 1 Hz periodic ISR NVIC — interrupt priority (15)
Design note: SysTick drives both scheduling decisions and the context switch directly, there is no PendSV deferral in this implementation. TIM2 provides an independent 1 Hz application-level interrupt. SysTick runs at NVIC priority 15 (lowest), ensuring it never preempts device driver ISRs.

Scheduler Execution Flow

SysTick fires every 10 ms (configurable via QUANTA). Both the scheduling decision and the register swap happen inside the SysTick ISR itself, no deferred interrupt. A thread can also trigger an early switch voluntarily via osThreadYield(), which resets the SysTick countdown and re-pends the interrupt immediately.

Context Switching

The Cortex-M hardware exception mechanism saves a partial frame automatically on interrupt entry (R0–R3, R12, LR, PC, xPSR). The SysTick handler manually saves and restores the remaining callee-saved registers (R4–R11), completing the full architectural context before swapping the stack pointer to the next thread.

/* SysTick handler — naked, no compiler prologue/epilogue */
__attribute__((naked)) void SysTick_Handler(void) {
    __asm("CPSID  I");                     /* disable interrupts */
    __asm("PUSH   {R4-R11}");              /* save callee-saved registers */
    __asm("LDR    R0, =currentPt");        /* R0 = &currentPt */
    __asm("LDR    R1, [R0]");              /* R1 = currentPt */
    __asm("STR    SP, [R1]");              /* currentPt->stackPt = SP */
    __asm("PUSH   {R0, LR}");
    __asm("BL     osSchedulerRoundRobin"); /* advance currentPt */
    __asm("POP    {R0, LR}");
    __asm("LDR    R1, [R0]");              /* R1 = new currentPt */
    __asm("LDR    SP, [R1]");              /* SP = next task stackPt */
    __asm("POP    {R4-R11}");              /* restore next task registers */
    __asm("CPSIE  I");                     /* enable interrupts */
    __asm("BX     LR");                    /* exception return → next task */
}
Note: All thread stacks use the MSP (Main Stack Pointer) in this implementation. A production kernel would separate kernel stack (MSP) from thread stacks (PSP) to prevent a thread stack overflow from silently corrupting the kernel.

Thread Control Block

Each thread is described by a TCB containing its saved stack pointer and a pointer to the next TCB in the circular linked list. The stack pointer is always the first field, this ensures zero-offset access from assembly without struct member arithmetic, a hard requirement for correct context switching.

typedef struct tcb {
    int32_t    *stackPt;   /* MUST be first — accessed directly from asm */
    struct tcb *next;      /* circular linked list pointer */
} tcbType;

Each thread is allocated a fixed stack of STACKSIZE (400) words = 1600 bytes. Before launch, each stack is pre-initialized with a fake exception frame so the first context switch returns cleanly into the thread function:

void osKernelStackInit(int i) {
    tcbs[i].stackPt = &TCB_STACK[i][STACKSIZE - 16];

    TCB_STACK[i][STACKSIZE - 1]  = (1u << 24);     /* xPSR: Thumb bit set        */
    TCB_STACK[i][STACKSIZE - 2]  = (int32_t)taskFn; /* PC: thread entry point     */
    /* R14–R4 pre-filled with 0xAAAAAAAA for debug visibility */
}
Critical: xPSR bit 24 (Thumb mode) must be set. Without it, the first exception return causes an immediate hard fault on Cortex-M.

Scheduler Design

The kernel is built on a single preemptive round-robin core. Periodic task execution and cooperative yielding are both supported, not as separate scheduler implementations, but as deliberate features layered on top of that one core. There is no mode switch; all three behaviors flow through the same SysTick interrupt and the same context switch path.

Behavior Implementation Trigger Notes
Round-Robin (base) osSchedulerRoundRobin() SysTick every 10 ms Advances currentPt through circular TCB list
Periodic (layered) Tick counter inside RR scheduler Every PERIOD scheduler ticks Callback fires in ISR context and must be short, non-blocking
Cooperative yield (layered) osThreadYield() Opt-in, called by thread Resets SysTick VAL and sets PENDSTSET, same switch path as preemption 
/* Periodic callback layered inside the round-robin scheduler */
void osSchedulerRoundRobin(void) {
    if ((++period_tick) == PERIOD) {
        (*task3)();          /* periodic callback fires every PERIOD ticks */
        period_tick = 0;
    }
    currentPt = currentPt->next;  /* round-robin advance always happens */
}

/* Cooperative yield — early trigger of the same SysTick mechanism */
void osThreadYield(void) {
    SysTick->VAL = 0;      /* reset countdown so next quanta is full */
    INTCTRL = PENDSTSET;   /* pend SysTick immediately via ICSR */
}

Semaphore Synchronization

The kernel provides counting semaphores for inter-task synchronization. All operations disable global interrupts to ensure atomicity. osSemaphoreWait uses a spin-wait that briefly re-enables interrupts on each iteration to allow SysTick to fire and switch to a thread that may signal the semaphore.

void osSemaphoreWait(int32_t *semaphore) {
    __disable_irq();
    while (*semaphore <= 0) {
        __enable_irq();    /* allow SysTick to fire and switch context */
        __disable_irq();
    }
    *semaphore -= 1;
    __enable_irq();
}

void osSemaphoreSet(int32_t *semaphore) {
    __disable_irq();
    *semaphore += 1;
    __enable_irq();
}

Example from main.c — task2 gates on semaphore2, opens the valve, then signals semaphore1 to unblock task1 and run the motor:

/* semaphore1 init=1, semaphore2 init=0 */

task2:  osSemaphoreWait(&semaphore2)  /* blocks until signalled */
        valve_open()
        osSemaphoreSet(&semaphore1)   /* unblocks task1 */

task1:  osSemaphoreWait(&semaphore1)  /* unblocked by task2 */
        motor_run()

Feature Breakdown

SysTick
10 ms Scheduling Tick

Configurable quanta via QUANTA define. SysTick runs the full scheduling decision and context switch in a single ISR, no deferred interrupt needed.

Context Switch
Naked ASM Handler

Declared __attribute__((naked)) so the compiler emits no prologue or epilogue. Register state is managed entirely by inline assembly, giving full control over the save/restore sequence.

TCB
Circular Linked List

Three fixed TCBs in a circular list. Stack pointer is the first field for zero-offset assembly access. Each stack pre-initialized with a fake exception frame including Thumb bit in xPSR.

Semaphores
Counting Semaphores

Atomic increment/decrement guarded by __disable_irq(). Spin-wait briefly re-enables interrupts each iteration to allow SysTick context switches while waiting.

TIM2
Independent 1 Hz ISR

TIM2 configured for a 1 Hz interrupt independent of the scheduler tick. Used for application-level periodic work without consuming a scheduler quanta.

UART
printf via USART2

USART2 on PA2 at 115200 baud. __io_putchar hooks into newlib's _write syscall stub so standard printf works from any thread.

Boot & Launch Sequence

The kernel takes control from main() immediately after thread registration. osSchedulerLaunch() manually unwinds the fake exception frame from the first TCB's stack and jumps into task0. From that point, SysTick drives all further scheduling indefinitely.

Build & Flash

The recommended workflow is STM32CubeIDE, which bundles the ARM toolchain, build system, and ST-Link flashing. The STM32F411 Nucleo has a built-in ST-Link, no external debugger needed.

01

Clone the repository

git clone https://github.com/mervinnguyen/m_core_rtos.git
02

Import into STM32CubeIDE

Open STM32CubeIDE, go to File → Import → General → Existing Projects into Workspace, click Browse and select the cloned folder. The project will appear with its checkbox ticked — click Finish.

Note: The folder must contain a .project and .cproject file for CubeIDE to recognise it as an Eclipse project.
03

Build

Right-click the project in Project Explorer and select Build Project, or press Ctrl+B (Windows/Linux) / Cmd+B (macOS). Confirm the Console ends with Build Finished — 0 errors. The .elf binary is written to the Debug/ folder.

04

Flash and run

Connect the Nucleo board via USB. Go to Run → Debug Configurations → STM32 Cortex-M C/C++ Application, confirm the Debugger tab shows ST-LINK (OpenOCD) with interface set to SWD, then click Debug. CubeIDE will flash the binary, reset the board, and halt at main(). Press F8 or Resume to start full execution.

Tip: To flash without the debugger attached, use Run → Run instead of Run → Debug.

Requirements: STM32CubeIDE (includes arm-none-eabi-gcc and OpenOCD) · STM32F411 Nucleo · USB Mini-B cable
