M-Core RTOS

// A Real-Time Operating System built from scratch on ARM Cortex-M

A ground-up implementation of a preemptive real-time operating system targeting ARM Cortex-M MCUs. Implements round-robin, cooperative, and periodic schedulers with PendSV-based context switching, a thread control block subsystem, and binary semaphore synchronization, built directly on top of SysTick and NVIC hardware primitives.

System Architecture

The system is structured as four distinct layers. Each layer has a single responsibility and communicates only with the layer directly below it — a strict dependency hierarchy that mirrors production-grade embedded OS design.

Design Principle: SysTick, PendSV, and NVIC are the three hardware primitives the entire kernel is built on. Every scheduling decision ultimately flows through these three peripheral interfaces.

Scheduler Execution Flow

Each scheduler tick is fully deterministic. SysTick fires at a fixed 1 ms interval, increments the system counter, triggers the scheduler decision, and defers the actual register swap to PendSV, the lowest-priority interrupt on the system, ensuring context switches never interrupt critical ISRs.

The separation of scheduling logic (SysTick) from the stack swap (PendSV) is a critical Cortex-M design pattern since it allows the scheduler to run at a higher priority than the context switch, guaranteeing that interrupt latency is never inflated by task register saves.

Context Switching

The Cortex-M hardware exception mechanism saves a partial frame automatically on interrupt entry. The PendSV handler saves the remaining callee-saved registers manually, completing the full architectural context before switching the stack pointer to the next thread.

/* PendSV handler — naked function, no prologue/epilogue */
__attribute__((naked)) void PendSV_Handler(void) {
    __asm volatile (
        "MRS     R0, PSP            \n"  /* load PSP of running task */
        "STMDB   R0!, {R4-R11}      \n"  /* push callee-saved regs   */
        "LDR     R1, =currentTCB    \n"
        "LDR     R2, [R1]           \n"
        "STR     R0, [R2]           \n"  /* save new SP into TCB     */
        "BL      os_scheduler       \n"  /* select next task         */
        "LDR     R2, [R1]           \n"
        "LDR     R0, [R2]           \n"  /* restore SP from next TCB */
        "LDMIA   R0!, {R4-R11}      \n"  /* pop callee-saved regs    */
        "MSR     PSP, R0            \n"
        "BX      LR                 \n"  /* exception return          */
    );
}

Thread Control Block

Each thread is fully described by its TCB. The stack pointer is always the first field — this ensures zero-offset access from assembly without any struct member arithmetic, a requirement for correct context switching from the PendSV handler.

typedef enum {
    THREAD_READY   = 0x00,
    THREAD_BLOCKED = 0x01,
    THREAD_SLEEP   = 0x02,
} thread_state_t;

typedef struct tcb {
    uint32_t        *stack_ptr;   /* MUST be first — accessed from asm */
    uint32_t         priority;    /* scheduler priority (0 = highest)  */
    thread_state_t   state;       /* READY | BLOCKED | SLEEP           */
    uint32_t         delay_ticks; /* tick count until wake             */
    struct tcb      *next;        /* circular linked list pointer      */
    const char      *name;        /* debug identifier                  */
} tcb_t;
Critical: Each thread stack must be pre-initialized with a fake exception frame before os_launch(). The PC field is set to the thread function address; the xPSR Thumb bit (bit 24) must be set. Without this, the first exception return crashes immediately.

Feature Breakdown

SysTick
1 ms System Timebase

Configurable tick frequency drives the scheduler. Global volatile uint32_t sys_tick counter provides consistent time reference for sleep and timeout logic across all threads.

PendSV
Deferred Context Switching

Context switches are deferred to PendSV which is the lowest-priority exception on Cortex-M. Guarantees ISR latency is never inflated by task register saves mid-interrupt.

TCB
Thread Control Blocks

Circular linked list of TCBs. Each entry tracks stack pointer, state, delay counter, and priority. Stack-pointer-first layout enables zero-offset assembly access.

IPC
Binary Semaphores

Counting semaphore implementation protects shared resources (UART, GPIO banks). Blocking sem_wait transitions the calling thread to BLOCKED without busy-waiting.

NVIC
Interrupt Priority Control

PendSV configured at lowest NVIC priority (255). SysTick at highest. Ensures the context switch never preempts device driver ISRs or time-critical interrupt handlers.

UART
Runtime Observability

UART logging layer provides scheduler trace output. Task switches, semaphore acquisitions, and sleep events can be emitted over serial without impacting timing behavior.

Boot & Launch Sequence

The RTOS launch takes control from main() immediately after thread registration. It initializes the first task's PSP, drops to unprivileged thread mode, and triggers the first context switch at which point the kernel never returns to main().
